生成器

发表于 2016-05-09   |   分类于 编程   |  

First lets understand iterators. According to Wikipedia, an iterator is an object that enables a programmer to traverse a container, particularly list. However, an iterator performs traversal and gives access to data elements in a container, but does not perform iteration. You might be confused so lets take it a bit slow. There are three parts namely:

  • Iterable
  • Iterator
  • Iteration

All of these parts are linked to each other. We will discuss them one by one and later talk about generators.

Iterable

An iterable is any object in Python which has an __iter__ or a __getitem__ method defined which returns an iterator or can take indexes. In short an iterable is any object which can provide us with an iterator.

Iterator

An iterator is any object in Python which has a next(python2) or __next__ method defined. That’s it. That’s an iterator.

Iteration

In simple words it is the process of taking an item form something e.g a list. When we use a loop to loop over something it is call iteration. It is the name given to the process itself. Now as we have a basic understanding of these terms let’s understand generators.

Generators

Generator are iterators, but you can only iterate over them once. It’s because they do not store all the values in memory, they generate the values on the fly. You use them by iterating over them, either with a ‘for’ loop or by passing them to any function or construct that iterates. Most of the time generators are implemented as functions.
